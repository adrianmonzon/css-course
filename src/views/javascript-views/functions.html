Funciones

  • Son objetos
  • Argumentos vs Parámetros:
    • Parámetros: variables que se establecen entre paréntesis al CREAR la función. (ej. sayHi(name);)
      • una función puede tener asignado un parámetro por defecto por si no recibe dicho argumento. Ej.:
      • function example(number = 3, secondNumber) { ... }
    • Argumentos: valor concreto entre paréntesis al LLAMAR a la función. (ej. sayHi('Juan');)
      • Al ser invocada la función, puede no recibir todos los argumentos (el valor de los ausentes sería undefined)

    Una función puede no recibir parámetros en su declaración y sí argumentos en su invocación

    • mediante el array arguments, podemos asignar cada argumento añadido en la invocación a la posición que queramos. Ej.:
    • function example() {
      console.log(arguments[0]); // 1 (lo que saldría al invocar a la función abajo)
      console.log(arguments[1]); // 2
      console.log(arguments[2]); // 3
      }
      example(1, 2, 3)
  • Métodos vs Funciones:
    • Método: función almacenada dentro de un objeto. Ejs.:
      • const example = { name: 'Max', greet: function sayHi() { console.log('Hi'); } }
      • document.addEventListener() (método del objeto document)
    • Función: sin almacenar dentro de un objeto
  • Hoisting: Al igual que sucede con las var (ver aquí), las funciones declaradas en JS se "elevan" en el contexto global y por eso pueden ser invocadas antes de su propia declaración. Ej.:
    • saludar()
      function saludar() {
      console.log('Hola')
      }
  • Callbacks
    • Def.: función que se le pasa como argumento a otra función
    • Ejs.:
      • array.forEach(el => console.log(el.name))
      • array.map(el => console.log(el.name))
