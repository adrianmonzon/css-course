Diseño responsive

Premisas:

  • Mobile first

    • Desarrollamos la web primeramente enfocados en la pantalla de un móvil.
    • Los usuarios ya suelen acceder desde los móviles o tablets antes que desde ordenadores.

  • Tope: unos 600px (40rem)

    • A partir de ahí, ya consideramos que la pantalla es de un dispositivo superior a un móvil o tablet.
    • Ejemplo:

      @media (min-width: 40rem) { .product-overview h1 { font-size: 3rem; } }

      Cuando el ancho de un dispositivo sea igual o mayor que 40rem (unos 600px), aplicamos más tamaño a la fuente (ya que estaríamos aplicando a dispositivos superiores.)

  • Media queries

    • Importante: para un correcto funcionamiento, tienen que ir ordenadas de menor a mayor valor en la hoja de estilos. Ejemplo con el min-width:

      @media (min-width: 40rem) { .product-overview h1 { font-size: 3rem; } }
      @media (min-width: 60rem) { .product-overview h1 { font-size: 3rem; } }

    • mydevice.io : comprobar tamaño de dispositivos (ver aquí)
    • En el código, lo correcto es colocarlas todas abajo del todo.
    • Ejemplo con la página de inicio

      Código por defecto (mobile First):

      .plan { background: #d5ffdc; text-align: center; padding: 1rem; margin: 0.5rem 0; width: 100%; }

      Código para Desktop:

      @media (min-width: 40rem) { .plan__list { width: 100%; text-align: center; } .plan { display: inline-block; width: 30%; vertical-align: middle; } }
    • Operadores lógicos:

      • and

        Para que se cumplan dos condiciones.

        Ejemplo con mínimo de ancho y dispositivo en horizontal (ej. iPad girado):

        @media (min-width: 40rem) and (orientation: landscape) {}

      • ,

        Es como un "or". Con que se cumpla una condición, vale.

        Ejemplo con mínimo de ancho o dispositivo en vertical:

        @media (min-width: 200rem), (orientation: portrait) {}

        Estando el dispositivo en vertical (portrait), ya se aplicaría.
